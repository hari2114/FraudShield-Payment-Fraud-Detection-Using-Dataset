// Detection Pipeline
How It Works

Every payment passes through a 4-stage pipeline in under 100 milliseconds.

01

Data ingestion

Transaction data hits the API — amount, merchant, device fingerprint, IP, location, timestamp, and user history.

02

Feature extraction

200+ features computed: velocity, geographic anomaly, time-of-day pattern, merchant risk score, device trust level.

03

ML scoring

XGBoost ensemble model outputs a fraud probability 0–100. Trained on 50M+ labeled transactions.

04

Decision & action

Score <35 → approve. 35–69 → step-up auth (OTP). 70+ → block and alert the user.

// Signal Types
What We Monitor

Six categories of signals feed the detection model continuously.

💰

Amount anomaly

Transactions far above your normal spend pattern are flagged.

  • Above 3× average spend
  • Round-number probing amounts
  • Micro-transactions (card testing)
📍

Geographic signals

Location mismatches and impossible travel are among the strongest fraud indicators.

  • Impossible travel detection
  • High-risk country transaction
  • IP vs billing address mismatch
⏱

Temporal patterns

When a transaction happens matters — fraud peaks between 1–5 AM.

  • Unusual hour for this user
  • Rapid burst of transactions
  • Weekend / holiday anomaly
🏪

Merchant risk

Certain merchant categories carry inherently higher fraud rates.

  • Crypto / gambling / luxury
  • First-time merchant
  • Merchant fraud history score
📱

Device & identity

Device fingerprinting catches fraudsters using stolen credentials.

  • Unknown device / browser
  • VPN / proxy / Tor exit node
  • Emulated device signals
🔄

Behavioural history

Your historical pattern is your fingerprint — deviations are suspicious.

  • Prior decline then approval
  • Category never used before
  • Account age vs amount ratio
